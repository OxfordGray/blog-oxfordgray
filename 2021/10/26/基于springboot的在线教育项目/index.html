开发过程中遇到的错误:

1.在编写讲师模块的时候，第一次启动测试出现问题：

报错信息

错误原因：数据源出现错误

疑点：

日志中显示8080端口，但是我的配置文件中明明填写了8001端口和URL

猜测：配置文件未生效

解决：配置文件必须名为:application.properties才能自动装配，不能随意起名

更改配置文件名字后，启动成功

2.swagger配置后无限弹窗，打不开页面

一番检查发现代码没有出现错误，重启了一下idea，问题解决了

缘故：每次更新maven项目的pom文件要clean一下缓存

3.因为idea异常关闭，端口被占用

原因：因为idea异常的关闭，上次启动的程序未能杀死，再次打开程序后端口被占用

解决办法：

打开cmd输入netstat -ano

找到被占用的端口，记住UID，用任务管理器结束它

4.npm下载依赖失败

原因
因为npm7.x对某些事情比npm6.x更严格。通常，最简单的解决方法是将–legacy-peer-deps标志传递给npm(e.g.，npm i –legacy-peer-deps），或者使用npm@6。

如果这不能立即起作用，也许可以先删除node_modules和package-lock.json。它们将被重新创建。

解决了上面那个错误又报了一个错误，原来是安装node-sass的时候依赖缺少

因为node-sass需要调用操作系统的一些东西，所有需要环境的支持。提示需要安装python2.7环境

npm install --global windows-build-tools 之后完美解决

5.前端调用后端接口出现问题

当使用get方法请求数据的时候，后端没法用requestbody注解接收到数据，此时要把前端和后端的请求方式都改成post，用json来传输数据

6.在写课程列表的时候，怎么写都一直弹这个错误

经过多次添加断点，添加console.log发现

原因：表格显示需要数组包含对象的形式，每个对象是一行数据，拿到的数据格式不对。

解决措施：确保拿到的表格数据显示类型为数组。

7.在写阿里云视频点播服务的时候，由于某个jar包未开源，要手动用maven下载，但是输入maven指令后报错

原因：需要权限

解决：用管理员权限打开 不要用右键打开

8.阿里云点播报错

原因:阿里云要用到BASE64Encoder这个类，只有jdk8之前能用，jdk8以后的版本用不了

解决：替换回jdk8

9.redis连接报错

注意到下列语句DENIED Redis is running in protected mode because protected mode is enabled, no bind address was specified

可能的原因有以下几条：

1.云服务器没有放行6379端口

2.安全模式未设置CONFIG SET protected-mode no

3.设置完没重启redis服务

解决

10.Handler dispatch failed； nested exception is java.lang.NoClassDefFoundError: javax/xml/bind/**

原因：是由于缺少jaxb-api包导致的。在Java 8以及以前的版本，是默认包含jaxb这个jar包的；但是在java SE 9.0就不再包含着个包了，如果使用则需要手动引进。

解决：手动导包

11.Unexpected token o in JSON at position 1

刚写完登录页面token转换，弹出这个页面

原因:因为被转换的值不符合JSON格式而造成的，在对对象赋值的时候自动调用了对象toStrong()导致传递cookie变成了[object Object]，所以在赋值的时候先加上JSON.stringify方法避免toString的调用

解决办法：

在设置cookie的时候用JSON.stringify方法

12.org.springframework.http.converter.HttpMessageConversionException: Type definition error

原因:忘记添加getset方法了

解决办法：在实体类上加上@Data
